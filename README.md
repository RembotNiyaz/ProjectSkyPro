# Документация тестов проекта
## Общая информация
Данный тестовый набор предназначен для проверки корректности работы модулей проекта, реализующих взаимодействие с внешними API, работу с моделями данных (самолетами), сохранение данных в файлы, а также пользовательский интерфейс. Тесты покрывают основные сценарии использования и обеспечивают проверку соответствия реализации заявленным требованиям.

# Структура тестов
## 1. Тестирование API-модулей (test_api.py)
Цель: Проверка корректности работы классов для взаимодействия с внешними API: OpenStreetMapAPI и OpenSkyAPI.
Что тестируется:
Правильность подключения к API (статус ответа).
Правильность получения данных о границах страны (get_country_bounding_box).
Получение данных о самолетах в указанной области (get_airplanes_in_bbox).
Примеры сценариев:
Мокирование ответов API для проверки логики обработки данных.
Проверка обработки ошибок при неверных ответах (например, статус не 200).
## 2. Тестирование модели Airplane (test_models.py)
Цель: Проверка корректности создания, сравнения и сериализации объектов Airplane.
Что тестируется:
Валидация входных данных при создании.
Магические методы сравнения (__lt__, __gt__, __eq__).
Метод to_dict() для сериализации.
Примеры сценариев:
Создание объектов с корректными и некорректными данными.
Сравнение самолетов по скорости и высоте.
Проверка сериализации в JSON-словарь.
## 3. Тестирование файловых операций (test_file_savers.py)
Цель: Проверка работы класса JSONFileSaver.
Что тестируется:
Создание файла при отсутствии.
Добавление нового самолета (без дублирования).
Удаление самолета по call_sign.
Получение всех данных.
Примеры сценариев:
Добавление самолета и проверка его наличия.
Попытка добавить дубль — отсутствие дублирования.
Удаление самолета и проверка отсутствия.
## 4. Тестирование утилит (test_utils.py)
Цель: Проверка функций фильтрации и сортировки.
Что тестируется:
filter_by_country() — фильтрация по странам регистрации.
get_top_n() — получение топ N самолетов по высоте.
Другие вспомогательные функции.
Примеры сценариев:
Передача тестовых данных и проверка правильности результата.
Проверка сортировки по высоте.
## 5. Тестирование интерфейса (test_main.py)
Цель: Проверка функции user_interaction() с использованием моков.
Что тестируется:
Правильная обработка пользовательских вводов.
Вызов методов API и файловых операций.
Корректный вывод и сохранение данных.
Примеры сценариев:
Мокирование input() для автоматического тестирования.
Проверка вызовов методов и правильности результата.

# Рекомендации по запуску тестов
Убедитесь, что все зависимости установлены.
Перед запуском убедитесь, что файлы, используемые тестами, очищены или настроены правильно.
Запускайте тесты командой:
CopyRun
pytest tests/
или

CopyRun
python -m pytest tests/
