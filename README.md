# Описание
Этот код реализует простую объектно‑ориентированную модель для работы с товарами (Product) и категориями товаров (Category). Подходит для базового учёта продукции в интернет‑магазине или складской системе.

# Основные классы
Product
Представляет отдельный товар.

# Атрибуты:

name — название товара (строка);

description — описание товара (строка);

price — цена (число с плавающей точкой);

quantity — количество на складе (целое число).

Category
Представляет категорию товаров.

Атрибуты класса (статические):

category_count — общее количество созданных категорий;

product_count — общее количество товаров во всех категориях.

Атрибуты экземпляра:

name — название категории (строка);

description — описание категории (строка);

products — список объектов Product, входящих в категорию.

При создании новой категории автоматически обновляются счётчики:

Category.category_count увеличивается на 1;

Category.product_count увеличивается на количество товаров в категории.

# Пример использования
Код демонстрирует:

Создание объектов Product (товары: смартфоны, телевизор).

Создание объектов Category (категории: «Смартфоны», «Телевизоры»).

Вывод основных атрибутов товаров и категорий.

Проверку значений счётчиков категорий и товаров.

# Как запустить
Сохраните код в файл main.py.

Выполните команду:

bash
python main.py
Ожидаемый вывод
Программа выведет:

Информацию о каждом товаре (название, описание, цену, количество).

Данные о категориях (название, описание, количество товаров).

Значения счётчиков:

Category.category_count (общее число категорий);

Category.product_count (общее число товаров во всех категориях).

# Ограничения и возможные улучшения
Текущие ограничения:

Нет валидации входных данных (например, проверка на отрицательную цену/количество).

Список товаров в категории можно изменять извне без контроля.

Счётчики не учитывают удаление товаров/категорий.

Возможные улучшения:

Добавить методы для добавления/удаления товаров из категории.

Реализовать валидацию данных в __init__.

Добавить метод для получения общей стоимости товаров в категории.

Использовать свойства (@property) для контроля доступа к атрибутам.

Добавить документацию (docstrings) к методам и классам.
